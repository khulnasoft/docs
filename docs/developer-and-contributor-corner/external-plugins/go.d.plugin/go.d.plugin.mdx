# go.d.plugin

`go.d.plugin` is a [Khulnasoft](https://github.com/khulnasoft/khulnasoft) external plugin:

- **Independent Operation**: Runs as a separate process from Khulnasoft core, visible in system process lists (`ps fax`).
- **Automated Management**: Integrated with Khulnasoft's lifecycle management, managed automatically by Khulnasoft (start/stop operations).
- **Efficient Communication**: Uses a unidirectional pipe for optimal data transfer to Khulnasoft.
- **Modular Architecture**:
    - Supports an unlimited number of data collection modules.
    - Each module can run multiple collection jobs simultaneously.
    - Easy to extend with new collection modules

### Required Linux capabilities

All capabilities are set automatically during Khulnasoft installation using the [official installation method](/docs/khulnasoft-agent/installation/linux).

| Capability          |                                                Required by                                                |
|:--------------------|:---------------------------------------------------------------------------------------------------------:|
| CAP_NET_RAW         |      [Ping](https://github.com/khulnasoft/khulnasoft/tree/master/src/go/plugin/go.d/collector/ping#readme)      |
| CAP_NET_ADMIN       | [Wireguard](https://github.com/khulnasoft/khulnasoft/tree/master/src/go/plugin/go.d/collector/wireguard#readme) |
| CAP_DAC_READ_SEARCH | [Filecheck](https://github.com/khulnasoft/khulnasoft/tree/master/src/go/plugin/go.d/collector/filecheck#readme) |

## Configuration

Edit the `go.d.conf` configuration file using `edit-config` from the
Khulnasoft [config directory](/docs/khulnasoft-agent/configuration#the-khulnasoft-config-directory),
which is typically at `/etc/khulnasoft`.

```bash
cd /etc/khulnasoft # Replace this path with your Khulnasoft config directory
sudo ./edit-config go.d.conf
```

Configurations are written in [YAML](http://yaml.org/).

- [plugin configuration](https://github.com/khulnasoft/khulnasoft/blob/master/src/go/plugin/go.d/config/go.d.conf)
- [specific module configuration](https://github.com/khulnasoft/khulnasoft/tree/master/src/go/plugin/go.d/config/go.d)

### Enable a collector

To enable a collector you should edit `go.d.conf` to uncomment the collector in question and change it from `no`
to `yes`.

For example, to enable the `example` plugin you would need to update `go.d.conf` from something like:

```yaml
modules:
#  example: no 
```

to

```yaml
modules:
  example: yes
```

Then [restart khulnasoft](/docs/khulnasoft-agent/maintenance/service-control) for the change to take effect.

## Troubleshooting

Plugin CLI:

```sh
Usage:
  orchestrator [OPTIONS] [update every]

Application Options:
  -m, --modules=    module name to run (default: all)
  -c, --config-dir= config dir to read
  -w, --watch-path= config path to watch
  -d, --debug       debug mode
  -v, --version     display the version and exit

Help Options:
  -h, --help        Show this help message
```

To debug specific module:

```sh
# become user khulnasoft
sudo su -s /bin/bash khulnasoft

# run plugin in debug mode
./go.d.plugin -d -m <module name>
```

Change `<module name>` to the [module name](#available-modules) you want to debug.
